Overview

Return to Top

The Value Added Components provide alternative entry points for applications to leverage the OMM-based ETA and are intended to simplify and compliment the use of the ETA. The Value Added Components allow the application developer to choose and customize the balance between ultra high performance raw access and ease of use feature functionality. The choice to leverage these components is up to the application developer; Value Added Component use is not required to use ETA. In addition, the application can utilize some or all of the Value Added Components. Value Added Components are written to the ETA interfaces and are designed to work alongside ETA. As a result, the interfaces have a similar look and feel to ETA interfaces in order to provide simple migration and consistent use between all Value Added Components and ETA.

Product Features

Return to Top

The ETA Valued Added Components contain the following components:

  • ETA Reactor Component
    • This is a callback-based component that provides connection management and event processing features for OMM-based applications.
    • Supports OMM-based Consumers, Providers and Non-Interactive Providers.
    • Provides domain-specific callbacks for the Login, Source Directory and Dictionary RDM representations.
  • Administrative Domain Model Representations
    • This component provides RDM-specific representations of the OMM Administrative domain models.
    • Provides classes and interfaces that represent specific messages defined in the ETA RDM Usage Guide document for the Login, Source Directory and Dictionary domains.
    • In addition, this component handles encoding and decoding of the OMM Administrative domain messages, which significantly reduces the amount of code necessary to interact with those domain models.
  • Value Added Utilities
    • This component is a collection of common classes, mainly used by the ETA Reactor.
