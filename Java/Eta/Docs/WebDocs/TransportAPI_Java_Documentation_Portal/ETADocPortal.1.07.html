What are Value Added Components?
The Value Added Components provide alternative and simpler entry points for applications to leverage the OMM-based Transport API Java Edition and are intended to simplify and compliment the use of the Transport API. The Value Added Components allow the application developer to choose and customize the balance between ultra high performance raw access and ease of use feature functionality. The choice to leverage these components is up to the application developer; Value Added Component use is not required to use the Transport API. In addition, the application can utilize some or all of the Value Added Components. Value Added Components are written to the Transport API interfaces and are designed to work alongside the Transport API. As a result, Value Added Component interfaces have a similar look and feel to Transport API interfaces to provide simple migration and consistent use between all Value Added Components and the Transport API.
Value Added Components are offered alongside the native Transport APIs to maximize user experience by allowing developers to more easily and rapidly develop Transport API applications.
