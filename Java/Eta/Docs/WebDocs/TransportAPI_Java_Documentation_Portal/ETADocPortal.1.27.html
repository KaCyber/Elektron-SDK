Transport API Java Edition

Performance Tools
upajConsPerf
The upajConsPerf application measures performance of the Transport API transport, encoders, and decoders in consuming Level I Market Price content directly from an OMM provider or through the Thomson Reuters Enterprise Platform for Real-Time (TREP-RT).
To measure latency, the provider inserts a timestamp randomly in each burst of updates. The consumer then decodes the timestamp from the update to determine end-to-end latency. upajConsPerf also supports the measurement of posting latency.
upajNIProvPerf
The upajNIProvPerf application measures performance of the Transport API transport, encoders, and decoders in providing Level I Market Price content to the TREP-RT Advanced Data Hub (ADH).
The provider creates two categories of threads:
•
A main thread: This thread collects and records statistical information.
•
Provider threads: Each thread connects to an ADH and provides market data.
upajProvPerf
The upajProvPerf application measures the performance of the Transport API transport, encoders. and decoders in providing Level I Market Price content either directly to an OMM Consumer or to the TREP-RT.
You can configure the provider to provide updates at various rates. To measure latency, a timestamp is randomly placed in each burst of updates. The consumer then decodes the timestamp from the update to determine end-to-end latency.
upajTransportPerf
The upajTransportPerf application measures performance of the Transport API transport, using the different supported connection types with variable message sizes. upajTransportPerf allows for performance measurements using all Transport API transport types, including shared memory, reliable multicast, TCP/IP, and HTTP.
