Transport API Performance Tools
upajConsPerf
The purpose of this upajConsPerf application is to measure performance of the Transport API transport, encoders and decoders in consuming Level I Market Price content directly from an OMM provider or through the Thomson Reuters Enterprise Platform.
To measure latency, a timestamp is randomly placed in each burst of updates by the provider. The consumer then decodes the timestamp from the update to determine the end-to-end latency. upajConsPerf also supports measurement of posting latency.
upajNIProvPerf
The purpose of this upajNIProvPerf application is to measure performance of the Transport API transport, encoders and decoders, in providing Level I Market Price content to the Thomson Reuters Enterprise Platform Advanced Data Hub (ADH).
The provider creates two categories of threads:
•
A main thread, which collects and records statistical information.
•
Provider threads, each of which connect to an ADH and provide market data.
upajProvPerf
The purpose of this upajProvPerf application is to measure performance of the Transport API transport, encoders and decoders in providing Level I Market Price content either directly to an OMM Consumer or to the Thomson Reuters Enterprise Platform.
The provider may be configured to provide updates at various rates. To measure latency, a timestamp is randomly placed in each burst of updates. The consumer then decodes the timestamp from the update to determine the end-to-end latency.
upajTransportPerf
The purpose of this upajTransportPerf application is to measure performance of all Transport API transport types, using the different supported connection types with variable message sizes (including shared memory, reliable multicast, TCP/IP, and HTTP).
