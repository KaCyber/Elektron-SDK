Qos Timeliness Values
 
Table 70: QosTimeliness Values  
QOS Timeliness
DESCRIPTION
UNSPECIFIED
timeliness is unspecified. Typically used by QoS initialization methods and not intended to be encoded or decoded.
REALTIME
timeliness is real time: data is updated as soon as new data is available. This is the highest-quality timeliness value. In conjunction with a rate of QosRates.TICK_BY_TICK, real time is the best overall QoS.
DELAYED_UNKNOWN
timeliness is delayed, though the amount of delay is unknown. This is a lower quality than REALTIME and might be worse than DELAYED (in which case the delay is known).
DELAYED
timeliness is delayed and the amount of delay is provided in Qos.timeInfo. This is lower quality than REALTIME and might be better than DELAYED_UNKNOWN.
