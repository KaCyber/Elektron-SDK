Reactor Error Handling
The ReactorErrorInfo object is used to return error or warning information to the application. This can be returned from the various Reactor methods as well as part of a callback method. When returned directly from a Reactor method, this indicates that an error occurred while processing in that method. If returned as part of a callback method, this indicates that an error has occurred on one of the channels that the Reactor is managing.
The ReactorErrorInfo members are described in the following table.
CLASS MEMBER
DESCRIPTION
code
An informational code about this error. Indicates whether it reports a failure condition or is intended to provide non-failure related information to the user. Refer to Table 2 for details on the available codes.
error
The underlying error information from the Transport API. This includes a pointer to the Channel that the error occurred on, both a Transport API and a system error number, and more descriptive error text. The Error and its values are described in the Transport API Java Developers Guide.
location
Provides information about the file and line that the error occurred at. Detailed error text is provided via the Error portion of this object.
