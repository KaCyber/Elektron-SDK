Payload Cache
Applications can leverage Transport APIâ€™s OMM payload cache feature. Using the payload cache, an application can maintain a local store of the OMM container data it consumes, publishes, or transforms. The cache maintains the latest values of OMM data entries: container values update to reflect the most recent refresh and update message payloads whenever the application receives them. The Transport API retrieves data from the cache entry in the form of an encoded OMM container. The payload cache is independent of other Value Added components, and only requires the Transport API Data Package and Transport API Message Package. Only library and API header files are available for the cache component.
