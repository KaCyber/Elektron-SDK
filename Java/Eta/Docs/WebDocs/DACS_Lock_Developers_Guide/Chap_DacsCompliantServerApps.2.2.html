Overview
To permission at the subservice level, the item’s permissioning requirements must be available at locations other than just the source. For DACS to permission a source service below the service level, the source must:
1.
Create locks containing permissioning information, and
2.
Map entitlement codes in the locks to subservice names.
For subservice level permissioning, a DACS-compliant source server must:
•
Establish Subservice Names
•
Define Entitlement Codes
•
Create and Write Locks
•
Publish the Map
•
Provide a Mechanism to Read the Map and Supply it to the DACS Station
