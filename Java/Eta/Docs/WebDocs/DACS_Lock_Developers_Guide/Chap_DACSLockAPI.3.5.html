Compression of DACS Locks
To minimize physical size, DACS Locks are compressed based on the Binary Coded Decimal (BCD) algorithm. For example, the DACS Lock API must make a lock with the following PE requirements:

Figure 2. DACS Lock Array after Compression
By using the compression mapping in Table 7, the DACS Lock API creates the unsigned character array in Figure 2, which illustrates some of the advantages to using this compression:
•
The compression does not limit the maximum value of a PE, other than a possible machine maximum that can be manipulated easily with the software language (i.e., (232)-1 for an unsigned long).
•
BCD compression is simple and fast without resulting in excessive computation.
•
The compressed DACS Lock is smaller then the actual ASCII string if it were sent.
