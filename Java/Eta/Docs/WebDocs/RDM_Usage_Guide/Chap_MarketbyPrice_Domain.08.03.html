Market By Price Request Message Use
A Market By Price request message is encoded and sent by OMM consumer applications. The request specifies the name of an item in which the consumer is interested.
To receive updates, a consumer can make a “streaming” request by setting the RequestMsgFlags.STREAMING flag. If the flag is not set, the consumer is requesting a “snapshot,” and the final part of the refresh (i.e., the refresh has the RefreshMsgFlags.REFRESH_COMPLETE flag set) indicates all responses have been received for the snapshot. Updates may be received in either case if the refresh has multiple parts.
A consumer can pause an item to stop updates (if the provider supports such functionality). For more information, refer to the Transport API C Edition Developers Guide.
