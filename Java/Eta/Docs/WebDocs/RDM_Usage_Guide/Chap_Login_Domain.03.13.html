Login Refresh Message Payload
When a Login request message asks for connection configuration information (i.e., DownloadConnectionConfig = 1), a provider capable of supplying these details should respond with extended connection information as the RefreshMsg payload. This information can be useful for load balancing connections across multiple providers or ADS components. This could be set up in a manner where some well-known providers act solely as load-balancing servers, monitoring the load and state of other providers and directing consumers to less-loaded providers to handle the information exchange.
The extended connection information contains a list of other providers, along with connection and load-related information, and is formatted as a sorted Vector type, where each VectorEntry contains an ElementList. Each vector entry contains data specific to one provider. The summary data (an ElementList) contains information about the number of Standby providers to which the consumer should connect. If this value is non-zero, the consumer is expected to support Warm Standby functionality and connect to multiple providers.
The list should be sorted in order of best to worst choice.
