Posting
Posting offers an easy way for an OMM consumer application to publish content to upstream components which can then provide the information. This can be done off-stream using the Login domain or on-stream using any other non-administrative domain. Use PostMsg to post content to the system. This message can contain any OMM container type as its payload, but this often is an Msg. A provider indicates support for posting via the supportOMMPost Login attribute, as described in Section 3.3.1. For more information on posting, refer to the Transport API C Edition Developers Guide.
