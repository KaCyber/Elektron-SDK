Source Directory Info Filter Entry
The Info filter entry (Directory.ServiceFilterFlags.INFO, Directory.ServiceFilterIds.INFO) conveys information that identifies a service and the content it can provide. This includes information about provided domain types (e.g. Market Price, Market By Order), available QoS, and the names of any dictionaries required to parse the published content.
The Info FilterEntry should be present when a service is first added, and should not be changed as long as the service remains in the list.

The available Qualities of Service.
•
If the data comes from one source, there will usually be only one QoS.
•
If there are multiple sources, more than one QoS may be available.
Note the default.
•
If QoS is not provided, the Transport API assumes the service provides Realtime, Tick-By-Tick data.
•
If the Transport API receives a QoS value of Unspecified, the Transport API interprets it as a QoS of Realtime, Tick-By-Tick.
See the Transport API C Edition Developers Guide for more information about QoS use and handling.
