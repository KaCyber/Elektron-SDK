Dictionary Refresh Message Use
A Dictionary refresh message is encoded and sent by OMM Interactive and non-interactive provider applications and provides the consumer with the content of the requested dictionary.
 
Table 34: Dictionary Refresh Message Member Use  
COMPONENT
DESCRIPTION / VALUE
msgClass
Required. MsgClasses.REFRESH == 2
domainType
Required. DomainTypes.DICTIONARY == 5
state
Required. Contains stream and data state information for the dictionary stream.
qos
Not used.
seqNum
Optional. A user-specified, item-level sequence number which can be used by the application for sequencing messages within this stream.
groupId
Not used.
permData
Optional. Used if provided dictionary requires permissioning.
extendedHeader
Not used.
msgKey.serviceId
Required. This should be the serviceId of the service providing the dictionary.
msgKey.nameType
Not used.
msgKey.name
Required. Specify a msgKey.flags value of MsgKeyFlags.HAS_NAME. Populate msgKey.name with the name corresponding to the provided dictionary.
msgKey.filter
Required. The filter is populated with the verbosity of the provided dictionary. Possible verbosities are described in Table 31: Dictionary Request Message Member Use.
msgKey.identifier
Not used.
msgKey.attrib
Not used.
Payload
Required. The payload structure varies depending on the dictionary’s type. However, payload is typically an Series containing an ElementList, while the series summaryData indicates the specific dictionary type.
