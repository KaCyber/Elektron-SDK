Source Directory Group Filter Entry
The Group filter entry (Directory.ServiceFilterFlags.GROUP, Directory.ServiceFilterIds.GROUP) conveys item group status and item group merge information. Every item stream is associated with an item group as defined by the groupId provided with the itemâ€™s RefreshMsg or StatusMsg. If some kind of change impacts all items within the same group, only a single group status message need be provided. For more information on item group use and handling, see the Transport API C Edition Developers Guide.
If multiple Group FilterEntrys are received in a single FilterList, then they should be applied in the order in which they were received.

A status change to be applied to all items whose groupId matches the Group element. It is equivalent to sending an StatusMsg to each item.
The streamState is only allowed to be StreamStates.OPEN or StreamStates.CLOSED_RECOVER.
If you need to convey group status text or code information without changing the data state, use the value DataStates.NO_CHANGE.
