Market By Order Refresh Message Use
A Market By Order refresh message is encoded and sent by OMM interactive provider and OMM non-interactive provider applications. A Market By Order refresh may be sent in multiple parts. It is possible for update and status messages to be delivered between parts of a refresh message, regardless of whether the request is streaming or non-streaming.
Providers must set the RefreshMsgFlags.CLEAR_CACHE flag on the solicited RefreshMsg. For multi-part refreshes, the RefreshMsgFlags.CLEAR_CACHE flag must be set on the first part only.
