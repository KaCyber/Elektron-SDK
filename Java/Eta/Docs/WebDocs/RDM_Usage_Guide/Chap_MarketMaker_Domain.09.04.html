Market Maker Update Message Use
A Market Maker update message is encoded and sent by OMM interactive provider and OMM non-interactive provider applications.
The provider can send an update message to add, update, or remove market maker information.
 
Table 64: Market Maker Update Message Member Use  
COMPONENT
DESCRIPTION / VALUE
msgClass
Required. MsgClasses.UPDATE == 4
domainType
Required. DomainTypes.MARKET_MAKER == 9
updateType
Required. Indicates the general content of the update. Typically sent as one of the following:
•
UpdateEventTypes.UNSPECIFIED == 0
•
UpdateEventTypes.QUOTE == 1
seqNum
Optional. A user-specified, item-level sequence number which can be used by the application for sequencing messages within this stream.
conflationCount
Optional. If a provider sends a conflated update, this specifies to the consumer how many updates are in the conflation.
The consumer indicates interest in this information by setting the RequestMsgFlags.CONF_INFO_IN_UPDATES flag in the request.
conflationTime
Optional. If a provider is sending a conflated update, this specifies the time interval (in milliseconds) over which data is conflated.
The consumer indicates interest in this information by setting the RequestMsgFlags.CONF_INFO_IN_UPDATES flag in the request.
permData
Optional. If present, specifies permissioning information associated only with the contents of this update.
extendedHeader
Not used.
msgKey.serviceId
Optional (Required if RequestMsgFlags.MSG_KEY_IN_UPDATES was set on the request). Specifies the ID of the service providing the data.
msgKey.nameType
Optional (Required if RequestMsgFlags.MSG_KEY_IN_UPDATES was set on the request). Should match the nameType specified in the item’s request message (typically InstrumentNameTypes.RIC).
msgKey.name
Optional (Required if RequestMsgFlags.MSG_KEY_IN_UPDATES was set on the request). Specifies the name of the provided item.
msgKey.filter
Not used.
msgKey.identifier
Not used.
msgKey.attrib
Not used.
Payload
Required. The order book is represented by an Map, where each entry contains an FieldList which in turn contains information about a market maker.
